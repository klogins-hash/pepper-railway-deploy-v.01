## ü§ù How to Contribute

### Reporting Issues

**Before submitting:**
- Search existing issues to avoid duplicates
- Gather specific details about the problem
- Include reproduction steps if applicable

**Issue types:**
- **Bug reports**: Prompt behavior not matching documentation
- **Feature requests**: New capabilities or improvements
- **Research updates**: New ADHD/INFJ/voice AI research
- **Documentation**: Improvements to guides or examples

### Suggesting Enhancements

We welcome suggestions for:
- **Prompt optimizations** based on real-world usage
- **Response pattern improvements** for specific scenarios
- **New use case adaptations** (different industries, roles)
- **Metrics and measurement** improvements
- **Documentation clarity** enhancements

### Contributing Code/Content

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b feature/amazing-improvement`)
3. **Make your changes** with clear commit messages
4. **Test thoroughly** if modifying prompts
5. **Update documentation** to reflect changes
6. **Submit a pull request** with detailed description

## üìù Contribution Guidelines

### For Prompt Changes

**Testing requirements:**
- Test with at least 5 different query types
- Verify response length stays under 20 seconds
- Confirm tone modulation works appropriately
- Check that challenge protocol remains intact
- Ensure ADHD/INFJ optimizations aren't degraded

**Documentation requirements:**
- Update prompt changelog with changes
- Explain reasoning behind modifications
- Provide example interactions showing improvement
- Note any trade-offs or considerations

### For Research Contributions

**Quality standards:**
- Peer-reviewed sources preferred
- Clear citations with DOI/URL when available
- Relevance to ADHD combined type, INFJ, or voice AI
- Summary of key findings and implications

**Integration:**
- Add to research/sources.md bibliography
- Update research/adhd-infj-research.md if substantial
- Link to relevant documentation sections

### For Documentation

**Style guide:**
- Clear, concise language (ADHD-friendly)
- Examples for abstract concepts
- Proper markdown formatting
- Links to related sections
- Accessibility considerations

## üß† Special Considerations

### Neurodivergent Contributors

We recognize that:
- ADHD contributors may prefer async communication
- Detail-oriented contributions may take longer
- Executive function support in contribution process helps
- Various communication styles are valuable

**We accommodate:**
- Flexible contribution timelines
- Breaking large changes into smaller PRs
- Clear templates and structures
- Patient, supportive review process

### Privacy & Ethics

**Protected information:**
- Never share real user conversations without consent
- Anonymize any examples or case studies
- Respect HIPAA and privacy regulations
- No medical/health data in contributions
